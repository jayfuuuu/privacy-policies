Privacy Policy

Media Snag – Chrome Extension

Overview

Media Snag is a browser extension that detects video and audio streams (HLS, DASH, and direct media files) on webpages and allows users to download them. It operates entirely within the user's browser and does not transmit any data to external servers controlled by the extension developer. This privacy policy explains what data is accessed, how it is used, and how it is stored.

Personal Data

Media Snag does not collect any personally identifiable information (PII) such as names, email addresses, or device identifiers. The extension does not require account creation or login. No analytics, tracking, or telemetry data is collected.

Data Collection

The extension accesses and processes the following data during normal operation:

  • Network request metadata: The extension monitors HTTP response headers (such as content-type and content-length) to detect media files on the pages the user visits. This data is held in memory only and is never persisted to disk or transmitted externally.
  • Request headers: HTTP request headers associated with detected media are temporarily cached in memory for up to 10 minutes to facilitate downloads. They are automatically discarded after expiration.
  • Page metadata: The page title, Open Graph title, thumbnail image URL, and favicon URL are collected from the active tab to display contextual information in the side panel. This data is held in memory only and is cleared when the tab is closed or navigates to a different origin.
  • Download records: When the user initiates a download, a record containing the download ID, filename, status, and progress is stored locally on the user's device using Chrome's chrome.storage.local API.
  • User preferences: Settings such as the "Save As dialog" toggle are stored locally on the user's device using chrome.storage.local.

Data Processing

Data is processed in the following ways:

  • Media detection: The extension listens to network responses via the webRequest API and classifies media types based on response headers and URL patterns. This happens locally in the browser with no external communication.
  • Stream resolution: For HLS streams, the extension fetches and parses .m3u8 playlist files to extract segment URLs. For DASH streams, it fetches and parses .mpd manifest files. These fetches are made directly to the media server hosting the content — no data is routed through any intermediary.
  • HLS download and remuxing: When the user initiates an HLS download, the extension creates an offscreen document that fetches individual stream segments, remuxes them into MP4 format, and assembles them into a single file. The resulting file is offered as a browser download. All processing occurs locally in the browser.
  • Direct downloads: For non-streaming media, the extension triggers the download using Chrome's native downloads API without additional processing.
  • DRM detection: The extension monitors calls to navigator.requestMediaKeySystemAccess solely to detect whether a page uses DRM-protected content. This detection is passive — it does not block, modify, or interfere with the DRM handshake, and no encryption keys or license data are accessed or stored.

Third-Party Services

Media Snag does not integrate with any third-party APIs, analytics services, or external servers controlled by the extension developer. The only network requests made by the extension are to the media servers that host the content on the pages the user visits — these are the same servers the user's browser would normally communicate with.

Data Storage and Retention

  • Download records and user preferences are stored locally on the user's device using Chrome's chrome.storage.local API.
  • No data is sent to the extension developer or any server controlled by the extension developer.
  • Tab-specific data (detected media, page metadata) is held in memory only and is automatically cleared when the tab is closed or navigates to a different origin.
  • Cached request headers are automatically discarded after 10 minutes.
  • Download records are retained locally until the user manually clears them or uninstalls the extension.

Data Security

All data is stored locally on the user's device using Chrome's chrome.storage.local API and is not accessible to other extensions or websites. Network requests to media servers use the same connection security (HTTPS) as the original page. No data is stored on any external server controlled by the extension developer.

Data Sharing

The extension does not sell, share, or transfer user data to any third party. No data leaves the user's browser except for direct requests to media servers hosting the content the user chooses to download.

Permissions

The extension requests the following browser permissions:

  • webRequest: Required to monitor network responses and detect media files on webpages.
  • downloads: Required to trigger and manage file downloads.
  • sidePanel: Required to display the media detection and download interface in the browser sidebar.
  • storage: Required to persist download records and user preferences locally on the device.
  • tabs: Required to associate detected media with the correct browser tab and retrieve tab information.
  • webNavigation: Required to detect page navigations and clean up tab-specific data when users navigate away.
  • scripting: Required to inject a content script that extracts page metadata (title, thumbnail, favicon) for display in the side panel.
  • activeTab: Required to interact with the currently active tab.
  • offscreen: Required to create an offscreen document for fetching and remuxing HLS stream segments into downloadable files.
  • host_permissions (<all_urls>): Required to detect media requests on any webpage the user visits. The extension must monitor all URLs because media streams can be hosted on any domain or CDN. No data from these requests is transmitted externally.

User Control

  • Users can clear their download history at any time from the Downloads tab within the extension.
  • Users can cancel, pause, or resume active downloads at any time.
  • Uninstalling the extension removes all locally stored data.
